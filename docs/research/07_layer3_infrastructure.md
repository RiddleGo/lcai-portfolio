NVIDIA 将本层定义为 **AI factories（AI 工厂）**：

> They are not designed to store information. They are designed to **manufacture intelligence**.  
> （它们不是为存储信息而设计，而是为**制造智能**而设计。）

基础设施层 = **土地 + 电力 + 冷却 + 网络 + 芯片集群 + 算力调度** 的系统集成。IEA 卫星追踪：专用 AI 数据中心容量在 **18 个月内增长 3 倍以上**。

## 第 1 章 本质定义

### 1.1 AI 工厂 vs 传统数据中心

| 维度 | 传统 DC | AI 工厂 |
|------|---------|---------|
| 核心功能 | 存储、检索、托管 | **训练 + 推理**，实时生成智能 |
| 工作负载 | 虚拟机、数据库 | GPU 集群矩阵运算 |
| 网络 | 10/25 GbE 为主 | **400G/800G InfiniBand/RoCE** |
| 冷却 | 风冷为主 | **液冷**标配 |
| 功率 | MW 级 | **100MW–GW 级** campus 涌现 |
| 建设周期 | 12–18 月 | 受电力约束或 3–5 年+ |

### 1.2 本层包含的子系统

```
AI 工厂
├── 物理设施：土地、建筑、机电
├── 电力系统：800V 配电、UPS、BESS
├── 冷却系统：冷板液冷、CDU、冷却塔
├── 计算集群：GPU 服务器、NVL72 机柜
├── 网络：Leaf-Spine、光模块、DCI
├── 存储：并行文件系统、对象存储
└── 软件：Kubernetes、Slurm、云调度、MLOps
```

### 1.3 为何是独立一层

- Capex 规模已超 **7250 亿美元/年**（四大 Hyperscaler 2026，Statista）
- 需要 **跨行业整合**：建筑、电力、IT、网络
- 从「买服务器」升级为 **「建工厂」** 的系统工程

## 第 2 章 工作原理

### 2.1 AI 工厂如何「制造智能」

**训练阶段**：
```
海量数据 → 分布式 GPU 集群 → 梯度同步（AllReduce）→ 更新模型权重 → 产出 checkpoint
```

**推理阶段**：
```
用户请求 → 负载均衡 → GPU 推理实例 → token 流式输出 → 计费
```

### 2.2 关键术语

| 术语 | 含义 |
|------|------|
| **Hyperscaler** | 超大规模云厂商（AWS、Azure、GCP、Meta） |
| **Neocloud** | AI 专用云（CoreWeave、Lambda 等） |
| **Colocation** | 第三方托管数据中心 |
| **NVL72** | NVIDIA 72-GPU 液冷整机柜（GB200/GB300） |
| **800V DC** | 高压直流配电，降低 AI 机柜配电损耗 |
| **IaaS/PaaS/MaaS** | 基础设施/平台/模型即服务 |

### 2.3 全栈交付模式（NVIDIA GTC 2026）

NVIDIA 从「卖 GPU」演进为 **「卖 AI 工厂」**：
- Kyber Rack 800V 机柜
- Spectrum-X 网络
- AI Enterprise 软件栈
- 与 Foxconn/Quanta 等 ODM 联合交付 **turnkey AI factory**

### 3.0 基础设施链总览

```
土地/EPC → 园区电力/液冷(→第1层) → 机柜/UPS/Busbar/800V配电
     → AI服务器(GPU/昇腾集成) → 存储(SSD) → 电源模块(→电容)
     → 交换芯片/DPU → 800G光模块 → 光纤 → IDC/AIDC上架出租
```

#### 8.6.0 基础设施层最高壁垒（全球 vs 中国 A 股）

**全球**：Capex **7250 亿美元** 超级周期下，瓶颈为 **GPU 到货 + 电力 + 800G 光互连**（文档第 5 章）；Microsoft 称算力产能约束至 2026。

**中国 A 股映射**：不持有 AWS/Azure，核心在 **「智算园区 + AI 服务器 + 光模块」**——把芯片组装成可售算力，并把集群用高速光网络连起来。

```
芯片（GPU/昇腾）到货
        ↓
AI 服务器集成（机柜）  ← 浪潮、工业富联
        ↓
800G 光模块组网        ← 中际旭创（全球最紧环节之一）
        ↓
IDC 上架出租算力       ← 润泽科技等 AIDC
```

**与第 1 层分工**：液冷 **英维克** 已在能源层；本层 **不重复** 选纯液冷，聚焦 **算力集成 + 网络 + 园区**。

#### 8.6.1 五维准则（同前两层）

| 维度 | 权重 | 基础设施层考察什么 | 淘汰谁 |
|------|------|-------------------|--------|

| **② 护城河** | 25% | 光模块 **全球份额**；服务器 **智算集采**；IDC **大客户绑定** | 无规模代工、无上架率 |
| **③ 政策确定性** | 20% | 东数西算、超大规模智算集群、算力互联互通 | 纯「云计算概念」 |

| **⑤ 可交易性** | 10% | 龙头流动性 | ST、壳 |

#### 8.6.3 「最最最」标的

**🥇 中际旭创 (300308)** — 800G 光互连：AI 工厂的「神经网络」

| 项目 | 内容 |
|------|------|
| **对应壁垒** | 集群 Scale-out 网络；800G **供不应求**（文档 5 章） |
| **为什么最优** | 全球光模块 **龙头**；直接受益 Hyperscaler + 国内智算 Capex；单 MW 光模块价值量随 AI 上升 |
| **验证指标** | 800G/1.6T 收入占比、毛利率、北美+国内大客户 cap ex |
| **主要风险** | 价格竞争；硅光技术路线变化 |

**🥈 浪潮信息 (000977)** — AI 服务器：把芯片变成可售算力

| 项目 | 内容 |
|------|------|
| **对应壁垒** | GPU/昇腾 **集成交付**；「All in 液冷」智算机柜 |
| **为什么最优** | 国内 AI 服务器 **份额领先**；政府/互联网智算集采核心供应商；与第 2 层芯片层 **衔接** |
| **验证指标** | AI 服务器收入增速、液冷机柜占比、天池基地产能 |
| **主要风险** | 毛利率承压；客户 cap ex 波动 |

**🥉 润泽科技 (300442)** — AIDC 园区：算力上架 = 基础设施兑现

| 项目 | 内容 |
|------|------|
| **对应壁垒** | 枢纽 **土地+电力+机柜** 资源；大客户长协 |
| **为什么最优** | **纯 AIDC** 运营商；绑定头部互联网；上架率/合同负债可跟踪 |
| **验证指标** | 上架率、单 kW 租金、新园区交付、绿电占比 |
| **主要风险** | 单客户集中；区域电力政策 |

**增强 A：工业富联 (601138)** — 云端 AI 服务器 ODM 龙头，业绩规模大、验证性强

## 第 9 章 层间传导

### 向下依赖
- 芯片（GPU 数量）+ 能源（电力）= 工厂能否运转

### 向上支撑
- 模型训练/推理 **必须** 在本层执行
- 无足够算力，模型无法迭代

### 与应用层
- Agent 并发 ↑ → 推理实例 ↑ → 需扩建 AI 工厂
- SaaS 公司从「卖软件」转向「买算力 + 卖 AI 服务」
