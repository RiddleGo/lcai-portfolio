## 第 1 章 本质定义

### 1.1 什么是 AI 模型

AI 模型 = **从数据中学习的参数化函数**，给定输入可产生预测或生成输出：

```
输入（文本/图像/分子结构）→ 神经网络（数十亿~万亿参数）→ 输出（文本/标签/3D结构）
```

### 1.2 模型层包含什么

| 组件 | 说明 |
|------|------|
| **基础模型** | LLM、多模态、科学模型 |
| **训练框架** | PyTorch、JAX、TensorFlow |
| **推理引擎** | TensorRT、vLLM、TGI |
| **微调工具** | LoRA、RLHF、DPO |
| **数据** | 预训练语料、指令数据、RLAIF |
| **MaaS 平台** | API 托管、私有化部署 |

### 1.3 为何独立成层（相对 2019「技术层」）

- 模型本身成为 **可交易资产**（API 按 token 计费）
- 开源 vs 闭源形成 **平行生态**
- 训练一次大模型耗资 **数亿~数十亿美元**
- 模型能力直接决定应用层天花板

## 第 2 章 工作原理

### 2.1 Transformer 核心机制

2017 年 Google 提出 Transformer，核心是 **自注意力（Self-Attention）**：

```
对于每个 token，计算它与序列中所有 token 的相关性权重 → 加权聚合信息
```

**优势**：
- 并行计算（优于 RNN 顺序计算）
- 长距离依赖建模
- 可扩展至万亿参数（Scaling Laws）

**关键概念**：

| 概念 | 含义 |
|------|------|
| **预训练 Pre-training** | 海量无标注文本学习语言规律 |
| **微调 Fine-tuning** | 特定任务/行业数据适配 |
| **RLHF** | 人类反馈强化学习，对齐人类偏好 |
| **推理 Inference** | 部署后生成输出；每 token 消耗算力 |
| **Context Window** | 模型一次可处理的 token 长度 |
| **MoE** | Mixture of Experts，稀疏激活降低推理成本 |

### 2.2 训练 vs 推理 economics

| 阶段 | 成本结构 | 频率 |
|------|----------|------|
| **训练** | 一次性巨大 Capex（万卡 ×  weeks） | 每代模型 1 次 |
| **推理** | 持续 Opex（每 token 电费+折旧） | 每次用户交互 |

**DeepSeek 启示**（NVIDIA 官方引用）：高效架构 + 开源 → 降低推理成本 → **应用层爆发** → 反而推高底层算力总需求。

### 2.3 模型分类

| 类型 | 代表 | 用途 |
|------|------|------|
| **LLM** | GPT、Claude、Llama、DeepSeek | 文本、代码、推理 |
| **多模态** | GPT-4o、Gemini | 图文音视频 |
| **科学 AI** | AlphaFold、分子生成模型 | 生物、化学 |
| **具身/机器人** | RT-2、π0 | 感知-行动 |
| **行业专用** | 医疗、法律、金融模型 | 合规场景 |
| **推理模型** | DeepSeek-R1、OpenAI o 系列 | Chain-of-Thought |

## 第 9 章 层间传导

### 向下（模型 → 芯片/能源/基础设施）
- 更大模型 → 更多 GPU × 更长训练 → 更高电力
- DeepSeek 高效架构 → 单任务算力↓，但总推理量↑ → 总需求仍↑

### 向上（模型 → 应用）
- 模型能力是应用天花板
- 模型跨越「可用阈值」后（NVIDIA 2025–2026），应用 PMF 涌现

### 开源的特殊传导
- 开放 weights → 降低应用门槛 → 中小企业部署 → **推理算力需求分散化爆发**
