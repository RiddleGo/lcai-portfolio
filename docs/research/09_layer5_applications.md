## 第 1 章 本质定义

### 1.1 应用层做什么

将模型层的「智能能力」封装为 **用户可感知的产品、服务或商业模式**，并产生收入。

### 1.2 NVIDIA 三分法（官方）

| 类型 | 定义 | 代表 |
|------|------|------|
| **知识应用 Knowledge** | 软件中生成智能 | 药物研发、法律助手、客服、代码生成 |
| **机器应用 Machine** | AI 驱动机器 | 自动驾驶 |
| **具身应用 Embodied** | AI 嵌入物理实体 | 人形机器人、工业机器人 |

### 1.3 2026 年最重要的应用形态：Agent

**智能体 Agent** = 能 **感知 → 规划 → 调用工具 → 执行 → 记忆** 的 AI 系统。

恒生 + IEA + OFweek 共同强调：Agent 是从「聊天机器人」迈向 **实体经济** 的核心载体，也是 **能耗上升** 的新主因（多步推理 × tool call）。

## 第 2 章 工作原理

### 2.1 传统 AI 应用 vs Agent

| 维度 | 传统 AI 应用 | Agent |
|------|-------------|-------|
| 交互 | 单轮问答 | 多步任务分解 |
| 能力 | 生成文本 | 调用 API、数据库、代码执行 |
|  autonomy | 低 | 高（自主规划） |
| 能耗 | 较低 | **高**（IEA：数百~数千倍于简单文本） |
| 示例 | ChatGPT 聊天 | Cursor 编程、Devin 开发、企业流程自动化 |

### 2.2 Agent 架构

```
用户目标
    ↓
Planner（规划器）→ 分解子任务
    ↓
Tool Router → 选择工具（搜索/代码/CRM/交易）
    ↓
Executor → 执行并观察结果
    ↓
Memory → 上下文与长期记忆
    ↓
输出 / 循环直至完成
```

### 2.3 企业落地路径

```
POC（概念验证）→ 试点（单部门）→ 规模化（全公司）→ 产品化（对外销售）
         ↑                    ↑
    Gartner：大量 POC 失败    2026：B端盈利兑现年（OFweek）
```

### 2.4 关键术语

| 术语 | 含义 |
|------|------|
| **PMF** | Product-Market Fit，产品市场契合 |
| **Copilot** | 辅助人类，Human-in-the-loop |
| **Autopilot** | 高度自主执行 |
| **RAG** | Retrieval-Augmented Generation，检索增强生成 |
| **Vertical SaaS + AI** | 垂直行业软件嵌入 AI |
| **Token Pay** | 按 token/任务计费的新商业模式 |

#### 8.6.0 应用层最高壁垒（全球 vs 中国 A 股）

**全球**：NVIDIA 三分法下，**经济价值在应用层兑现**；恒生强调赢家需 **专属数据 + 模型能力 + 深度嵌入运营** 三者兼备。

**中国 A 股映射**：2027 年智能体普及率 **>70%**（国务院）；2026 为 **B 端盈利兑现年**（OFweek）。最高壁垒不是「会做 App」，而是：

```
Agent/行业 Know-how + 私有数据 + 嵌入核心工作流
        ↓
替换成本极高 → 马太效应（OFweek）
```

| 应用类型 | A 股映射方向 |
|----------|-------------|
| **知识应用** | 办公/金融/政务 Copilot |
| **机器应用** | 智能驾驶 |
| **具身应用** | 机器人核心部件/平台 |

**与第 4 层分工**：讯飞/金山在模型层已选；应用层 **不重复**，选 **行业 PMF 更清晰** 的标的。

## 第 9 章 层间传导（应用层的核心角色）

### 9.1 向下拉动（最重要）

```
应用 PMF → 推理请求 ↑ → 模型 API 收入 ↑ → 云厂商扩建 AI 工厂
          → 训练下一代模型 → GPU 订单 ↑ → 电力需求 ↑
```

**DeepSeek-R1 案例**（NVIDIA）：开源强推理模型 → 应用层采用加速 → 底层训练/芯片/能源需求同步上升。

### 9.2 向上反馈

- 应用需求定义模型能力优先级（长上下文、tool use、低延迟）
- 垂直场景数据 **回流** 模型微调

### 9.3 与 SaaS 生态的冲突

恒生 2026：Agent 渗透 **开发、维护、整合** → 传统 SaaS 按 seat 收费模式承压 → 利润向 **有数据+有模型** 的平台集中。

## 结语：五层闭环

应用层是五层架构的 **「温度计」** 和 **「发动机」**：

- **温度计**：应用有没有 PMF，决定下方投资是理性还是泡沫
- **发动机**：每一个成功的 Agent、Robot、Copilot，都在向下方发出 **采购订单**——从 API 调用到 GPU 到电力

NVIDIA：我们仍处于早期——大部分基础设施尚未建成，大部分机遇尚未实现。  
但方向已明确：**Every company will use AI. Every nation will build it.**
