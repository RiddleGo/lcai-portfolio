# LCAI 日常维护手册

> 知识库维护、新增股票、改规则的标准操作流程。  
> 更新日期：2026-06-07 · 网页查看：[资产总览 → 手册](../资产总览.html#handbook)

---

## 先理解：仓库里「知识库」是什么

LCAI 不是单一文件，而是 **多层数据** 协同工作：

| 层级 | 文件 | 谁维护 | 作用 |
|------|------|--------|------|
| **规则知识** | [`criteria.json`](criteria.json) | **网页「规则」Tab 提交** 或手工改 | 买卖裁决唯一标准；规则引用 `book_ids` |
| **书籍笔记** | [`../书籍/books/*.md`](../书籍/books/) | **网页「书籍」Tab 提交** 或手工改 | 每本书独立 Markdown + frontmatter |
| **书籍索引** | [`../书籍/books-index.json`](../书籍/books-index.json) | `build_books_index.py` 自动生成 | 网页筛选、规则 `book_ids` 校验 |
| **书籍目录页** | [`../书籍/投资书单.md`](../书籍/投资书单.md) | 脚本自动生成 | 人类可读目录，**勿手改表格** |
| **持仓清单** | [`../holdings.json`](../holdings.json) | **唯一源** | 股数、成本、secid；同步到网页与行情 |
| **持仓行情** | [`../quotes-data.js`](../quotes-data.js) | `fetch_quotes.py` 生成 | 现价 + LCAI「已持仓」判定 |
| **云端关注** | [`../watchlist-data.js`](../watchlist-data.js) | Issue / 脚本 | 每周一批量刷新报告 |
| **研判缓存** | [`../reports/{代码}/`](../reports/) | Actions / 脚本生成 | 网页秒开「综合研判」，**勿手改 JSON** |
| **持仓 UI 片段** | [`../holdings-data.js`](../holdings-data.js) | `sync_holdings.py` 生成 | 供 `资产总览.html` 读取 |

**关键区别：**

- 网页「⭐ 收藏」→ 只存浏览器 **localStorage**，换设备会丢
- 「☁️ 云端关注」→ 写入 `watchlist-data.js`，每周一自动刷新报告
- 「已持仓」判定 → `quotes-data.js` 里是否有该 `secid`（如 `1.601127`）
- 规则「来源」→ 由 `book_ids` + `meta_ids` 解析书名，**勿手填** `sources` 字符串

## 场景 A：只想研判一只新票（未买入）

### 方式 1 · 网页（推荐）

1. 打开 [资产总览 → 选股](../资产总览.html#screen)
2. 输入代码 → **「帮我看看」**
3. 常看 → **「⭐ 收藏并加入云端队列」** → GitHub Issue → **Submit**
4. Actions 约 2–5 分钟后生成 `reports/{代码}/` 并写入关注列表
5. 之后 **每周一** 自动刷新

### 方式 2 · 本地 CLI

```bash
python scripts/lcai_screen_json.py 600519          # 只看结论
bash scripts/run_lcai_analysis.sh 600519           # 写入 reports/
python scripts/watchlist_utils.py add 600519       # 加入云端关注（可选）
```

### 方式 3 · GitHub Actions

[Generate LCAI reports](https://github.com/RiddleGo/lcai-portfolio/actions/workflows/lcai-reports.yml) → **Run workflow** → 填 `symbol` 或留空（全量）

## 场景 B：真正买入 / 新增持仓

### 方式 1 · 网页（推荐）

1. 选股页 **「帮我看看」** 看完研判
2. 点 **「📥 加入持仓（网页提交）」**
3. 填写股数、成本价、账户 → **下一步 → GitHub Submit**
4. 等 2–5 分钟，持仓页与 LCAI「已持仓」分支自动生效

### 方式 2 · 编辑 `holdings.json`（高级）

**推荐：只改 [`holdings.json`](../holdings.json)，再跑同步脚本。**

### 1. 编辑 `holdings.json`

在 `holdings` 数组增加一条（示例）：

```json
{
  "id": "moutai",
  "name": "贵州茅台",
  "account": "hb",
  "symbol": "1.600519",
  "shares": 100,
  "costPerShare": 1500,
  "currency": "CNY",
  "hk": false,
  "fallbackPrice": 1500,
  "sold": false
}
```

- A 股 secid：`1.`（6/5 开头）或 `0.`（0/3 开头）
- 港股 secid：`116.` + 5 位代码

### 2. 同步到网页与行情源

```bash
python scripts/sync_holdings.py    # → holdings-data.js
python scripts/fetch_quotes.py     # → quotes-data.js（含新 secid）
```

### 3. 生成 LCAI 报告（含持有/减仓/卖出分支）

```bash
bash scripts/run_lcai_analysis.sh 600519
```

可选：`python scripts/watchlist_utils.py add 600519`，以便每周 `--all` 刷新。

### 4. 提交 Git

```bash
git add holdings.json holdings-data.js quotes-data.js reports/
git commit -m "add holding 600519"
git push
```

## 场景 C：卖出 / 移除股票

| 目的 | 操作 |
|------|------|
| 不再显示在持仓页 | `holdings.json` 中设 `"sold": true` → `sync_holdings.py` |
| LCAI 不再走「已持仓」 | 从 `holdings.json` 删除或 `sold: true` → `fetch_quotes.py` |
| 停止每周自动刷新 | 从 `watchlist-data.js` 移除代码 |
| 删除历史报告 | 删除 `reports/{代码}/`（可选） |

## 场景 D：规则 / 判定标准维护

### 网页编辑（推荐，不用懂代码）

1. 打开 [资产总览 → 规则](../资产总览.html#criteria)
2. 改数字：建仓线、各层权重、行业 PE、每条规则门槛
3. **参考书籍**：从多选下拉选书（写入 `book_ids`），禁止自由文本
4. 点 **「☁️ 提交到云端」** → 在新开的 GitHub 页点 **Submit**
5. 回到规则页等待约 5–10 分钟（会自动提示完成）

- **💾 保存草稿**：只存本浏览器，未提交前别人看不到  
- **↩️ 恢复线上版本**：放弃未提交的修改  

### 高级（开发者）

编辑 [`criteria.json`](criteria.json) → `python scripts/apply_criteria.py --reports`

规则分类见 [`rule-categories.json`](rule-categories.json)；非书籍来源见 [`meta-sources.json`](meta-sources.json)。

## 场景 F：书籍笔记 / 分类 / 关联规则

### 补一本书笔记

1. 打开 [资产总览 → 书籍](../资产总览.html#books)
2. 左侧筛选 → 选中一本书 → 右侧阅读 stub 正文
3. 本地直接改 `书籍/books/{book-id}.md`，或点 **「☁️ 提交到云端」** → GitHub Issue `[book]` → Submit
4. Actions 写入 md 并运行 `build_books_index.py`（**不会**自动重算全部报告）

### 给书加分类

编辑该书 frontmatter 的 `categories: [估值, 价值投资]`，保存后运行：

```bash
python scripts/build_books_index.py
```

### 把书关联到新规则

1. **规则 Tab**：在对应规则的多选下拉中加入该书（写入 `book_ids`）→ 提交 `[criteria]`
2. **书籍 frontmatter**：在 `related_rules: [L3-01]` 中补上规则编号（建议双向一致）
3. `validate_criteria.py` 会对不一致给出警告（非阻断）

**勿手改** [`投资书单.md`](../书籍/投资书单.md) 目录表格；改 `books/*.md` 或跑索引脚本即可。

## 场景 G：批量分析书籍 → 候选规则 → 合并进 criteria

三层文件：

| 层 | 文件 |
|----|------|
| 书摘 | `书籍/books/*.md`（status: analyzed / reviewed） |
| 候选 | [`book-rule-candidates.json`](book-rule-candidates.json)（110 条，每书 1 条） |
| 合并 | [`book-rules-merged.json`](book-rules-merged.json) + [`book-rules-merge-report.md`](book-rules-merge-report.md) |

### 1. 批量分析（AI + 可选网络）

```bash
# 单本 / 已挂规则的书（P0 校准）
python scripts/analyze_books_batch.py --book-id b019-9a3426
python scripts/analyze_books_batch.py --linked

# 全库（快速，跳过网络）
python scripts/analyze_books_batch.py --all --missing --no-web

# 带网络检索（较慢，需可访问外网）
python scripts/analyze_books_batch.py --all --missing

# 可选 LLM 增强：设置 OPENAI_API_KEY、OPENAI_BASE_URL
```

### 2. 去重合并（dry-run 先看报告）

```bash
python scripts/merge_book_rules.py --dry-run
python scripts/merge_book_rules.py
# 查看 投资系统/book-rules-merge-report.md
```

### 3. 审阅通过后写入 criteria

```bash
python scripts/apply_merged_rules.py --approve
# 等价于 validate + 摘要 + 引擎回归
```

合并策略：**优先扩展现有 25 条规则的 book_ids**，不自动改 threshold。当前 110 本已全部挂入规则出处。

## 快速决策表

| 你想… | 怎么做 |
|--------|--------|
| 看一只新票能不能买 | 网页「帮我看看」 |
| 常看 + 每周自动更新 | 收藏 → GitHub Submit 一次 |

| 补一本书笔记 | **书籍 Tab** → 提交 `[book]`；或改 `书籍/books/*.md` |
| 规则引用新书 | **规则 Tab** 多选书籍 → `[criteria]` 提交 |
| 批量书摘+规则 | `analyze_books_batch.py` → `merge_book_rules.py` → `apply_merged_rules.py --approve` |
